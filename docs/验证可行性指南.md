## 2. 制作静态 tile

需要将已有 Mars 数据转换为 WPS geogrid binary tile，而不是直接改文件名：

```text
MOLA/TES 原文件
  -> 解码字节序、维度、比例因子和缺测值
  -> 统一到 0..360 经度和南到北纬度顺序
  -> 转成 index.copy 所描述的 1 度网格，或更新 index 描述实际网格
  -> 写出类似 00001-00360.00001-00180 的 tile
```

全陆面与单一土壤分类 tile 可以人工构造为常数 1；物理字段不得在没有说明的情况下用地球数据代替。

## 3. 运行名称

WPS 实际读取的名称没有 `.copy` 后缀：

```text
namelist.wps
geogrid/GEOGRID.TBL
WPS_GEOG_MARS/<field>/index
geogrid.exe
```

因此验证应在单独运行目录中从 `.copy` 结果生成这些运行名称，不覆盖本目录保留的原文件。`geogrid.exe.copy` 是未进行火星半径修改的 WPS 4.6 参考二进制，不能代替重新编译的 Mars 版本。

## 4. 静态检查重点

- `namelist.wps` 中不应再有全角逗号或全角感叹号。
- `max_dom=1`、`e_we=73`、`e_sn=37` 应与 MarsWRF 单域和 MACDA 72 x 36 网格对应。
- intermediate 文件名、`prefix` 和 `fg_name` 都应使用 `MACDA`。
- `geo_em.d01.nc` 的 mass grid 应为 72 x 36，经度为 0..355，纬度为 -87.5..87.5。
- `HGT_M` 单位应为 m；`ALBEDO/ALBBCK` 和 `EMISS/EMBCK` 应为 0..1。
- 当前 real 兼容路径的 `ALBEDO12M` 应为百分数并有 12 层；不能把 0..1 数据交给随后除以 100 的代码。
- `THC/THCBCK` 数值和单位必须与 MarsWRF 实际读取逻辑核对。
- `LANDMASK` 应全为 1，不应引入地球海洋或地球土地利用类别。
- NetCDF 全局属性中的半径、`DX` 和 `DY` 应对应火星，不是 6370 km 地球半径。
- 时间属性应保留 `0028-00507_02:00:00`，不能被改回伪造的地球日期。

## 5. 下游检查重点

即使 geogrid 输出正确，也要检查：

- metgrid 是否认识五位 sol 并能调度 `MACDA:0028-00507_02`。
- `METGRID.TBL` 是否从 `PRESSURE` 派生 `PRES`，并传递 `TT/UU/VV/GHT/PSFC/SKINTEMP`。
- `real.exe` 是否把 `ALBEDO/ALBBCK`、`EMISS/EMBCK`、`THC/THCBCK` 带入 `wrfinput_d01`。
- `real.exe` 是否错误地对火星反照率再除以 100。
- `wrfinput_d01` 的 `PLANET_YEAR`、`P2SI`、`Times`、网格半径和静态字段是否一致。
