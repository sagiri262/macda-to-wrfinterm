## 2. namelist.wps 核心修改

| 配置 | 原文件 | `.copy` 结果 | 原因 |
|---|---|---|---|
| domain 数 | `max_dom=2` | `max_dom=1` | 当前 `em_global_mars` 和 MACDA 转换均使用单域 |
| 时间 | 分量字段且不完整 | `0028-00507_02:00:00` 至 `0028-00537_00:00:00` | 与 MACDA MY28/SOY507-537 和 MarsWRF HDATE 对齐 |
| 强迫间隔 | `21600` | `7200` | MACDA 文件为 2 个火星小时一条记录 |
| 投影 | `lambert` | `lat-lon` | 输入和目标都是火星全球规则经纬网格 |
| 网格 | 两层嵌套 | `e_we=73, e_sn=37` 单域 | 对应 72 x 36 mass grid 和 5 度间距 |
| `dx/dy` | `295826` | 省略 | WPS 的 lat-lon 分支中 `dx/dy` 单位为度；省略表示全球域 |
| 经度起点 | 地球 `stand_lon=-98` | `stand_lon=-2.5` | 让第一个 mass point 位于 0 度东经，与 intermediate 的 0..355 度一致 |
| 静态分辨率 | `10m,2m` | `mars_default` | 与 Mars 专用 GEOGRID.TBL keyword 对齐 |
| 静态路径 | 外部地球路径 | 项目内 `sample/WPS_GEOG_MARS` | 消除不可访问且不适用于火星的地球路径 |
| intermediate 前缀 | `MACDA_FILE`/`FILE` 不一致 | 均为 `MACDA` | 与转换器输出 `MACDA:0028-...` 对齐 |
| 输出格式 | NetCDF | NetCDF | 保留 `io_form_geogrid=2` 和 `io_form_metgrid=2` |
| 字符 | 含全角标点 | 纯 ASCII namelist | 避免 Fortran namelist 解析错误 |

## 3. GEOGRID.TBL 核心修改

旧表主要面向地球：土地利用、绿色植被比例、雪反照率、城市参数和地球土壤分类。新表只描述本项目需要的火星静态场：

| 字段 | 类型 | 数据目录 | optional | 说明 |
|---|---|---|---|---|
| `HGT_M` | continuous | `mola_topography/` | 否 | MOLA 地形，是最低必需静态场 |
| `LANDUSEF/LU_INDEX/LANDMASK` | categorical | `mars_surface_class/` | 否 | 全陆面分类；类别 2 仅保留为 water 类定义 |
| `SOILCTOP/SCT_DOM` | categorical | `mars_soil_class/` | 否 | 单一火星土壤占位分类 |
| `SOILCBOT/SCB_DOM` | categorical | `mars_soil_class/` | 否 | 同上 |
| `SOILTEMP` | continuous | `mars_deep_soil_temp/` | 否 | 当前 real 初始化使用的深层土温兼容场 |
| `GREENFRAC` | continuous，12 层 | `mars_greenfrac/` | 否 | 无植被火星表面，12 个月均为零 |
| `ALBEDO12M` | continuous，12 层 | `mars_albedo12m/` | 否 | 当前 real 路径使用百分数并除以 100 |
| `SNOALB` | continuous | `mars_snow_albedo/` | 否 | 当前 real 路径的兼容字段，第一版为零 |
| `ALBEDO/ALBBCK` | continuous | `mars_albedo/` | 是 | 必须为 0..1，不是百分数 |
| `THC/THCBCK` | continuous | `mars_thermal_inertia/` | 是 | 火星地表热惯量 |
| `EMISS/EMBCK` | continuous | `mars_emissivity/` | 是 | 0..1 发射率 |
| `H2OICE` | continuous | `mars_h2oice/` | 是 | 后续增强字段 |

沙尘 `TAU_OD2D` 和 MACDA `CO2ICE` 是时变 intermediate 字段，不放入静态 `GEOGRID.TBL`。

`optional=yes` 是 WPS 4.6.0 支持的语法。旧 `marswrf/WPS/geogrid/src/source_data_module.F` 不识别该选项，因此该表以 WPS 4.6 语法为目标。

## 4. index.copy 模板

每个数据目录都有独立 `index.copy`，共 11 份，包含投影、分辨率、已知点、字节数、tile 大小、类别范围、单位和描述。模板统一假定转换后的第一版产品为 1 度全球网格：

```text
projection=regular_ll
dx=1.0, dy=1.0
known_lat=-89.5, known_lon=0.5
tile_x=360, tile_y=180, tile_z=1
tile_bdr=3
```

这些值描述“计划生成的 WPS tile”，不是对现有 MOLA/TES 原文件的声明。若采用 1/64 度 MOLA 或其他分块策略，必须同步修改 index。

注意两套反照率单位不同：`ALBEDO12M` 是当前 `real.exe` 兼容路径，tile 中使用百分数；`ALBEDO/ALBBCK` 是 Mars 直接字段，使用 0..1。后续若修改 real 初始化逻辑，应只保留一条明确的数据路径，不能重复除以 100。
