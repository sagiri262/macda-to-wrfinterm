## ./geogrid.exe 程序

`geogrid` 的目的是定义模拟区域，并将各种陆地静态数据集插值到模式网格上。这些区域由用户在 `WPS namelist` 文件 `namelist.wps` 的 `&geogrid` 记录中指定的信息来定义。默认情况下，除了在每个网格点计算纬度、经度和地图尺度因子之外，`geogrid` 还会把土壤类别、土地利用类别、地形高度、年平均深层土壤温度、月植被覆盖度、月反照率、最大雪反照率以及坡度类别插值到模式网格上。每个字段的全球数据集都可以通过 WPS 地理静态数据下载页面获得；由于这些数据是时间不变的，因此只需要下载一次。有些数据集只有一种分辨率，而另一些则同时提供“全分辨率”和“低分辨率”版本。一般来说，低分辨率静态场数据只适合代码测试和教学用途；凡是关注模式精度的应用，都应使用全分辨率地理数据集。

除了插值默认陆地字段外，`geogrid` 程序也足够通用，可以把大多数连续型和类别型字段插值到模拟区域。新的或额外的数据集可以通过表文件 `GEOGRID.TBL` 插值到模拟区域中。`GEOGRID.TBL` 文件定义了 `geogrid` 将生成的每个字段；它描述了某个字段要使用的插值方法，以及该字段数据集在文件系统中的位置。

`geogrid` 的输出采用 `WRF I/O API` 格式。因此，如果选择 `NetCDF I/O` 格式，`geogrid` 就可以输出 NetCDF 文件，便于使用外部软件包进行可视化，包括 `ncview`、`NCL` 和 `RIP4`。
